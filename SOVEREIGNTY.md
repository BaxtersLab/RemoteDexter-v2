# Sovereignty Principles

RemoteDexter (RD) is a peer-to-peer system with user-owned endpoints. It does not depend on any third-party infrastructure.

## Core principles
- RD is peer-to-peer, not cloud-based.
- The User PC is the only endpoint the User Device connects to.
- RD never calls home to third-party infrastructure.
- Bootstrap is offline and user-approved.
- All connections are user-initiated and user-owned.

## Pairing boundaries
- A Paired Device is created only through explicit OS-level pairing.
- RD does not perform silent pairing or background device enrollment.

## Why this reduces systemic risk
- No centralized servers means no single point of failure.
- No call-home behavior reduces data leakage risk.
- User-owned endpoints reduce third-party trust requirements.

## Bootstrap overview
A Bootstrap Channel (Bluetooth, AirDrop, or USB) is used for initial installation or pairing. This process is offline and requires explicit user action on both devices.
